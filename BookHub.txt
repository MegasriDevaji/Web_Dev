BookHub - Online Book Store Structure
1. CUSTOMER FEATURES
A. Homepage & Navigation
Header with logo, search bar, shopping cart icon, user profile menu
Navigation menu (Home, Books, Categories, About, Contact)
Hero section with featured books carousel
Best sellers section
New arrivals section
Footer with links, contact info, social media
B. Book Browsing & Discovery
Browse Books Page - Display all books in grid/list view
Category Filter - Fiction, Non-Fiction, Science, History, Romance, etc.
Search Functionality - Search by title, author, ISBN (AJAX - no page refresh)
Advanced Filters - Price range, rating, publication year, language
Sort Options - By price, rating, newest, bestselling, alphabetical
Book Details Page - Title, author, description, price, rating, reviews, cover image, availability
C. Shopping Cart
Add/remove books from cart (AJAX)
Update quantity
Real-time price calculation
Cart summary (subtotal, tax, discount)
Apply coupon codes
Save cart (persistent storage)
Cart abandonment warning
D. Wishlist/Favorites
Add books to wishlist (requires login)
View saved books
Move from wishlist to cart
Share wishlist with others
E. User Authentication & Profile
Registration - Email, password, name, address
Login - Email/password authentication
Password Reset - Forgot password functionality
User Dashboard - Order history, saved addresses, payment methods
Edit Profile - Update personal info, change password
Address Book - Multiple delivery addresses
F. Reviews & Ratings
Display 5-star rating system
Customer reviews with text
Filter reviews by rating
Helpful/unhelpful votes
Only verified buyers can review
G. Checkout Process
Review cart items
Select delivery address (or add new)
Choose shipping method (Standard, Express, Overnight)
Review order summary
Mock payment processing
Order confirmation with order ID
H. Order Management
View order history
Track order status (Pending → Processing → Shipped → Delivered)
Download invoice/receipt
Cancel order (if not shipped)
Return/refund request
2. PAYMENT SYSTEM (Mock)
Payment Flow
Customer enters card details on checkout page
Frontend validates card format (AJAX)
Sends payment request to backend
Backend processes payment (simulates 500ms delay)
Generates transaction ID
Returns success/failure response
Displays confirmation page
Payment Data Stored
Transaction ID
Order ID (linked)
Amount paid
Payment date/time
Payment status (Success/Failed/Pending)
Card last 4 digits (masked)

5. TECHNOLOGY IMPLEMENTATION

JavaScript - Form validation, cart logic, DOM manipulation

Database
